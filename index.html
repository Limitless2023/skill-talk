Part 01

什么是 Skill

把 AI 想象成超级能干的全能实习生

✗ 没有 Skill

每次都要手把手教：
"标题要震惊、排版要分段、最后加二维码……"

—每次重复教，费口舌（费 Token）
—AI 容易忘，下次从零开始
—好的经验用完就丢

✓ 有了 Skill

写好操作手册塞进背包，喊一声就自动干活

›按需触发 — AI 自动识别何时该用
›不占记忆 — 不用时不消耗上下文
›可复用可分享 — 写一次，团队都能用

Part 02

Skill 长什么样

本质就是一个文件夹，非常轻量

my-contract-reviewer/
├── SKILL.md ← 大脑（指令 + 流程 + 触发条件）
└── scripts/ ← 手脚（Python/Bash 脚本）
    └── check_clauses.py

SKILL.md 里写什么

①身份证 — 名称 + 描述 + 触发条件
②执行指令 — 按什么步骤做
③输出格式 — 结果长什么样
④注意事项 — 边界和限制

设计原则

●按需触发 — 通过 description 匹配
●渐进加载 — 不用时不占上下文
●死板的活交给脚本 — 省 Token 又精准
⚠安全 — 第三方 Skill 务必审查代码

创建 Skill 5 步法

来自 Anthropic 官方教程

01
定义任务边界
明确 Skill 要解决什么问题、输入是什么、输出是什么。
例：审查广告投放协议 → 输出风险清单 + 修改建议
02
撰写 SKILL.md 指令
用自然语言写清楚：触发条件、执行步骤、输出格式、注意事项。
不需要代码！大白话就行，AI 看得懂
03
添加辅助脚本（可选）
需要精确计算、格式转换、API 调用？加个 Python/Bash 脚本。
不会写也没关系 — 让 AI 帮你写脚本
04
测试 + 迭代
拿真实案例跑一遍，看哪里不对就改指令。
关键：每次修正都让 Skill 变聪明一点
05
上线 + 分享
放进 Claude Projects / Skills 文件夹，开始用！
写一次，团队都能用。经验变成可复制的资产。
Part 02 续

Skill vs MCP 别搞混了

一个教方法，一个给数据。两者结合 = 超级大厨

Skill = 菜谱 教怎么做

关注How — 思考逻辑、做事步骤
形态几个 Markdown 文件
优势极轻量，打包就能分享
局限没法实时获取数据

MCP = 冰箱 给食材/数据

关注What — 连接外部数据和工具
形态运行的服务（Server）
优势能力强大，连接万物
局限配置门槛高

实战：写一份股市分析报告

MCP抓今天股价新闻 （食材）
Skill按巴菲特逻辑分析 （烹饪）
Skill生成 PDF 报告 （摆盘）
MCP发邮件给老板 （外卖送达）

未来 AI 工作流：MCP 连接数据资产 + Skill 固化业务流程

市场的真实问题

大企业的困境不是"缺技术"

—缺的不是技术 — 大模型能力摆在那里，工作流平台也很成熟
—缺对能力边界的清醒认识 — AI 能做什么、不能做什么

Part 04

专家知识到底是什么

不是"资料很多"，不是"懂行业术语"

步骤拆解 SOP

专家知道按什么顺序做，为什么这么做

01主体信息是否完整、匹配
02关键权责与违约边界
03付款与交付条件
04争议解决条款
05风险兜底与补救路径

✅ 完成标准 验收口径

专家知道什么算"合格"、什么算"有风险"

●字段缺失 → 不能出报告
●数值异常 → 触发风险提示
●结论必须保守表达

没有验收口径 → 凭感觉 → 项目必崩

Part 05

为什么专家知识比技术更关键

技术往往不是第一难点

✗没专家 → 流程拆不出 → 模型"自由发挥" = 不可控、不可交付
✗没标准 → 没法评估 → 60分还是95分？错一次损失多大？
✗没业务 → 提示词写不出 → 像不会做菜的人写菜谱

有技术但没专家 = 有发动机但没方向盘，车再好也开不动

Part 06

有了专家也未必好使

两个隐藏的大坑

!

坑一：专家不愿意配合

把能力交给 AI = 稀释自己在公司的稀缺性

"老子花了钱，你就得把事儿办好，不要总是麻烦我。"
— 企业客户真实心态
!

坑二：专家讲不清楚

"知识的诅咒" — 做太久，判断已内化成直觉

"为什么先看这个字段？" → "我就是习惯先看"
"什么情况算风险？" → "看多了就知道"

大量时间花在"帮专家把经验翻译成可执行的步骤"上

Part 07

横着切还是竖着切？
以合同审查为例

同样是"合同审查 Agent"，路径不同，结果天差地别

横着切 通用审查

SOP只能写通用检查项
准确率60-70%
结果能用，但不敢信
本质Demo 级

竖着切 广告协议专审

SOP精准到每个关键条款
准确率95%+
结果敢用，能信，可交付
本质生产级

竖着切的专家知识长这样

①投放平台条款 vs 框架协议冲突
②效果对赌的风险敞口
③数据回传合规（个保法+GDPR）
④结算周期 & 违约金条款

建议：先竖后横 — 选一个高频高价值场景做到 95%+，再复制到下一个

"落地的本质不是写代码，
而是把专家脑子里的隐性经验，
翻译成机器能执行的显性规则。"

谁能完成这个翻译，谁就掌握了 Agent 落地的核心能力

Part 10

Skill 的局限：80 → 95 的鸿沟

快速做到 80 分很容易，但最后 15 分才是真正的战场

80 分 天级

标准化流程SOP 明确的重复任务
格式化输出报告、摘要、周报
信息提取从合同中抽字段

最后 15 分 月级

边界案例非标合同、混合条款
幻觉控制编造不存在的引用
一致性同一份审两次不同
01

与现有系统集成

数据散落在飞书合同、OA、邮件、共享文档 — AI 怎么串？每个对接都是脏活累活

02

持续迭代

85% → 收集错误 → 优化 SOP → 90% → 加边界规则 → 93%… 没反馈就永远停在 80

03

人机协作边界

全自动出事故就被叫停；全人工没效率。按风险分流：低风险自动，高风险转人工

务实建议

01先跑再优化 — 80 分今天就能用 02人工兜底 — AI 初筛，人终审 03反馈循环 — 每次修正都是迭代燃料 04量化 ROI — 用数据说话
